Detailed Description

The abstraction for a node in the network.

You must create at most one node per thread. You can have many nodes in one process by creating several threads.

Examples:
client_server/client_node.cpp, client_server/server_node.cpp, client_server_msg/client_node_msg.cpp, client_server_msg/server_node_msg.cpp, client_server_oop/client_node_object.cpp, client_server_oop/server_node_object.cpp, publisher_subscriber/publisher_node.cpp, publisher_subscriber/subscriber_node.cpp, publisher_subscriber/subscriber_node_object_manual.cpp, publisher_subscriber_msg/publisher_node_msg.cpp, publisher_subscriber_msg/subscriber_node_msg.cpp, publisher_subscriber_multi/multi_publisher.cpp, publisher_subscriber_multi/multi_subscriber.cpp, publisher_subscriber_oop/publisher_node_object.cpp, publisher_subscriber_oop/subscriber_node_object.cpp, remapping/const.cpp, remapping/operation.cpp, and remapping/print.cpp.

Parameters
nodeNamethe name of the node

getName()

Create a node with a given name.

A message will be send to resolver to announce this node presence on the network. If a node with the same name already exists in the network, this node will get a different name.

The heartbeat thread will be started to let resolver track the online status of this node.

Examples:
client_server_oop/server_node_object.cpp, publisher_subscriber/subscriber_node_object_manual.cpp, publisher_subscriber_oop/publisher_node_object.cpp, and publisher_subscriber_oop/subscriber_node_object.cpp.

Run the spin loop.

This will continuously call spinOnce() and the specified callback, at the specified rate.

If the spinRate parameter is not specified, the value returned by b0::Node::getSpinRate() will be used.

Parameters
callbacka callback to be called each time after spinOnce()
spinRatethe approximate frequency (in Hz) at which spinOnce() will be called

◆ spinOnce()

virtual void b0::Node::spinOnce ( )
virtual

Read all available messages from the various ZeroMQ sockets, and dispatch them to callbacks.

This method will call b0::Subscriber::spinOnce() and b0::ServiceServer::spinOnce() on the subscribers and service servers that belong to this node.

Warning: every message sent on a topic which has no registered callback will be discarded.

Reimplemented in b0::resolver::Resolver.

Examples:
publisher_subscriber_msg/publisher_node_msg.cpp, publisher_subscriber_oop/publisher_node_object.cpp, and remapping/const.cpp.
